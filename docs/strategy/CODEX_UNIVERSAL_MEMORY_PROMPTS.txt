Prompt B3: Add Persistent Shared-Scope Support

Implement the first database and persistence changes needed for true shared memory scopes.

Deliverables:
- Add migrations
- Add persistence-layer code
- Add tests for default isolation, explicit sharing, conflict handling, and migration safety
- Create or update docs/strategy/shared-memory-scopes.md
- Update docs/strategy/schema-migration-plan.md and execution-log.md

Requirements:
- Introduce first-class persistence for namespace, client/app, agent, session, and project/repo scope.
- Make it possible for multiple clients and agents to intentionally share a namespace while preserving isolation for unrelated scopes.
- Keep defaults safe and unsurprising: legacy/single-project behavior remains isolated unless sharing is explicitly configured.
- Sharing must be explicit opt-in; no implicit cross-namespace or cross-project reads.
- Define and implement conflict behavior for concurrent writes in shared scopes (deterministic merge/coexistence rules, no silent data loss).
- Define privacy boundaries precisely, including namespace boundaries, project boundaries, and visibility of agent/session metadata.
- Preserve existing installations through an idempotent migration path with rollback-safe failure behavior.
- Add migration regressions for upgrade from existing installs and for unchanged legacy project-only calls.
