Universal Memory Prompt Pack (Operational)

Global rules:
- Preserve trust invariants: temporal correctness, provenance traceability, contradiction visibility, drift auditability, scope isolation.
- Keep canonical behavior in service/client layers; adapters should delegate.
- Update docs and tests in the same change when behavior changes.
- Prefer additive migrations with compatibility defaults.

Prompt 1: Scope/Policy changes
"Implement scope or policy enhancements without breaking existing default behavior. Add schema + service + adapter + test updates in one pass."

Prompt 2: Query semantics changes
"Change canonical query semantics first, then propagate to Python/MCP/REST/OpenAI surfaces and add parity tests."

Prompt 3: Trust layer changes
"For claim/provenance/contradiction/drift changes, include migration safety, event integrity checks, and novelty metric impact assessment."

Prompt 4: Adapter integration work
"Add one external adapter path end-to-end using existing canonical semantics. Do not fork semantics per adapter."

Prompt 5: Release readiness
"When trust-impacting behavior changes, run novelty eval + release gate verification and report artifact paths."

Definition of done:
- Code changes complete.
- Tests and linters pass.
- Docs updated and code-aligned.
- Adapter parity preserved.
- Evidence generated when required.
